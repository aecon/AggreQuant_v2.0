The Challenge

Our Screen

A genome-wide CRISPR activation screen for α-synuclein aggregation: 65 plates imaged in duplicate (130 plate runs), each a 384-well microplate with 3 channels per field of view:

  • Nuclei — DAPI stain (390 nm)
  • Cells — FarRed marker (640 nm)
  • Aggregates — GFP-tagged α-synuclein (473 nm)

The Bottleneck

  • Multi-software pipelines — chaining Fiji + ilastik + CellProfiler introduces compatibility issues and intermediate file proliferation
  • Filter-based parameters — thresholds and filter sizes must be re-tuned per batch/plate; not generalizable across experiments
  • No integrated quality control — out-of-focus fields get processed and corrupt downstream statistics
  • Reproducibility — different operators choose different parameters, producing different results on the same data

AggreQuant — The Starting Point

Automated HCS Image Analysis

AggreQuant was built to process our genome-wide CRISPR activation screen. For each field of view, it corrects uneven illumination, segments nuclei, cells, and aggregates, then applies colocalization to quantify aggregate-positive cells.

It successfully processed all 130 plate runs of the screen.

Segmentation Approach

  • Nuclei — StarDist pre-trained deep learning
  • Cells — Cellpose, or distance-intensity watershed
  • Aggregates — conventional image processing filters

Limitations

  • Manual configuration — parameters tuned per dataset
  • No input validation, no focus quality control
  • Single fixed UNet, no systematic model comparison
  • No GUI — edit setup.yml, run python main.py
Processing pipeline: preprocessing → segmentation → colocalization → quantification

The Workflow

  1. Manually edit applications/setup.yml with paths, channel patterns, plate layout, and control wells
  2. Hardcode the YAML path inside main.py:
    dataset = Dataset("applications/setup.yml")
  3. Run python main.py
  4. Repeat for every plate — edit the YAML, change the path, re-run

Pain Points

  • One config file per tray — 65 plates × 2 replicates = 130 manual edits
  • No validation — a typo in channel name silently produces wrong results
  • Control wells typed by hand — 32 well IDs per plate, easy to mistype
  • Plate layout hardcoded — rows, columns, fields must match exactly
  • No error messages — failures appear deep in processing, not at config time

The YAML configuration file is now generated automatically by the GUI and CLI.

Serves as a backup & reproducibility record — contains all settings: image analysis parameters, algorithms chosen, thresholds, quality metrics, and output options.

Every field validated at load time through typed dataclasses:

  • ChannelConfig — channel names, patterns, purposes
  • SegmentationConfig — algorithm parameters
  • QualityConfig — focus metric selection
  • OutputConfig — output paths and flags
  • PipelineConfig — top-level orchestration

Errors caught before processing — no more debugging halfway through a plate run.

Three Interfaces new

  • CLI — python run_pipeline.py config.yaml
  • Web GUI — Dash-based browser interface with batch queue, live logs, and result tabs
  • Python API — SegmentationPipeline(config).run()

Filter-Based

Classical image processing — no training data needed

Pipeline:

Intensity cap → BG normalize → Threshold → Median filter → Components → Size filter
  • Fast (< 0.1 s/image), fully interpretable
  • Configurable threshold via YAML
  • Good for high-contrast, well-separated aggregates

UNet Deep Learning

PyTorch — trained on 19 annotated fluorescence images

Supports:

  • 7 architecture variants (baseline, ResUNet, Attention, SE, ASPP, ConvNeXt)
  • 7 loss functions (BCE, Dice, DiceBCE, Focal, Tversky, …)
  • Full-resolution or tiled inference (192×192, Gaussian blending)

Why Does Class Imbalance Matter?

Most pixels are background. A naive model could predict “no aggregate” everywhere and still be >95% accurate. The loss function must force the network to care about the rare foreground pixels — but how much?

Too much weight on foreground → high recall, but many false positives (hallucinated aggregates)
Balanced weight → fewer false positives, but might miss faint aggregates

Training Setup

Adam optimizer (lr=1e-3) • ReduceLROnPlateau (halves lr after 10 stagnant epochs) • early stopping (patience 20, max 200 epochs) • augmentation (flip, rotate, affine, gamma, noise)

What We Compared

4 loss configurations varying the foreground weight (pos_weight 7.5 vs 3.0) and whether to add a Dice overlap term. Same architecture (baseline UNet, 31M params) for all runs.

Loss Function Effect

BCE (pw=7.5)

Each pixel scored independently. Foreground errors cost 7.5× more than background.

Pushes hard for recall — “don’t miss anything” — but tolerates many false positives.

Used in AggreQuant v1 (TensorFlow)

BCE (pw=3.0)

Same idea, but foreground only 3× more important. Rebalances the recall-precision trade-off.

Single change: +0.08 Dice, +0.15 precision, −0.07 recall vs pw=7.5.

DiceBCE (0.3 / 0.7, pw=3.0)

Adds Dice loss (overlap ratio) to BCE. Dice penalizes false positives structurally — every extra pixel hurts the score.

BCE-heavy mix (0.7) limits Dice’s sensitivity to noisy annotation boundaries.

Loss Function — Metrics Comparison

How Certain Is the Network When It’s Wrong?

Most FPs and FNs Are on Aggregate Edges

Annotation Uncertainty at Edges

Aggregate boundaries are inherently ambiguous — annotators are uncertain about where the aggregate ends and the background begins.

This means edge FPs and FNs are not true errors — they reflect annotation noise, not model failure.

  • 71% of UNet FPs are edge pixels — boundary disagreements with the annotator
  • 91% of FNs are also on edges — the model drew a slightly tighter boundary
  • Only 29% of FPs are core (hallucinated) — many of these are likely annotation misses

Reported Dice (0.81) likely underestimates true model performance.

Deep Learning vs Filter-Based Segmentation

Why Compare?

Classical filters detect aggregates by thresholding pixel intensity above a local background. This is fast and intuitive — but filters have no concept of shape. They draw boundaries wherever intensity drops, regardless of whether that’s the true edge of an aggregate or just noise.

The result: filters over-segment — they make aggregates look bigger than they are, merge nearby ones, and hallucinate objects in noisy regions. This distorts every downstream measurement: area, count, shape, eccentricity.

A deep learning model (UNet) learns what aggregates actually look like from annotated examples. Can it do better?

Experiment: Same 19 annotated images, evaluated with the same metrics. UNet (DiceBCE, pw=3.0) vs classical filter (threshold 1.60).

DL vs Filter — Visual Comparison

Conclusions

  • UNet Dice 0.83 vs Filter 0.58 — wins on every metric except raw recall (+2%)
  • Morphology: UNet median area 32 px (GT: 30 px); Filter inflates ~4× (122 px)
  • Objects: UNet 18,549 vs GT 19,524; Filter merges → only 10,415
  • Filter FPs: 1.2M px (37% hallucinated); UNet: 193K px (71% edge-only)

Filter-based profiling is unreliable — UNet is essential for morphological measurements.

Note: GT was created by correcting filter predictions — filter recall is artificially inflated. Despite this bias, UNet still wins.

UNet Architecture & Building Blocks

Encoder-Decoder Core

DoubleConv — Two 3×3 convolutions with BatchNorm and ReLU. The fundamental building block of the original UNet.

ResidualBlock — Adds a skip connection (identity or 1×1 conv) around the DoubleConv. Enables deeper networks without degradation.

ConvNeXt Block — Depthwise separable 7×7 conv + LayerNorm + GELU. Modern pure-convolution design (Liu 2022).

ASPP Bridge — Atrous Spatial Pyramid Pooling at the bottleneck. Captures multi-scale context through parallel dilated convolutions (rates 6, 12, 18).

Attention Mechanisms

Attention Gate — Learns to suppress irrelevant skip connection features using decoder context. Focuses on task-relevant structures (Oktay 2018).

SE Block — Squeeze-and-Excitation. Global average pool → FC → per-channel scaling. Learns which channels are most informative (Hu 2018).

CBAM — Convolutional Block Attention Module. Combines channel attention (like SE) with spatial attention (where to look).

ECA — Efficient Channel Attention. Lightweight alternative to SE using 1D convolution instead of FC layers (Wang 2020).

  • Complete, modular pipeline from raw HCS images to per-cell quantification
  • Three segmentation stages: StarDist nuclei, Cellpose cells, UNet/filter aggregates
  • Deep learning platform with 7 architecture variants and 7 loss functions
  • Focus quality assessment for automated image rejection

Key Scientific Findings

  • UNet achieves Dice 0.83 vs filter 0.58 for aggregates

  • Architecture ablation: only 1.1% Dice span across 7 variants — data, not architecture, is the bottleneck
