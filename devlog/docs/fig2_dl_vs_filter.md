## Annotation bias

The ground truth masks were generated by **manual correction of filter-based
predictions**. This introduces a systematic bias:

1. **Filter object recall is artificially inflated.** Annotators started from
   the filter output and corrected boundaries, so the GT inherently contains
   nearly every object the filter detected. The filter's near-perfect object
   precision (~0.95–0.99) reflects this — it's finding what it originally
   proposed.

2. **UNet object recall is penalized.** Any aggregate the filter missed was
   also likely missed by the annotator (not in the GT). The UNet may detect
   some of these, but they would count as FP rather than TP. Conversely, if
   the UNet misses aggregates that the filter+annotator caught, those count
   as FN.

3. **Filter precision is the honest metric.** The low precision (~0.40–0.58)
   is real — it measures how much the annotator had to trim back the filter's
   over-segmentation. The gap between filter prediction and corrected
   annotation is the actual over-segmentation.

4. **Despite this bias, UNet achieves higher Dice and precision.** The UNet
   learns tighter boundaries than the filter, even though the GT was derived
   from filter predictions. This makes the UNet's advantage conservative —
   its true performance is likely better than reported.

**For the paper:** Acknowledge the annotation protocol in Methods:
"Ground truth masks were generated by manual correction of filter-based
predictions, which may bias object-level recall in favor of the filter
method."
