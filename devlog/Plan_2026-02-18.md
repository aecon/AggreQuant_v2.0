### 5. Focus metric heatmaps вЂ” one plate heatmap per computed focus metric

**Goal**: Automatically generate plate heatmaps for every focus quality metric in the
CSV, so the user can visually inspect per-well focus quality at a glance.

#### Which columns to plot

Focus columns in `field_measurements.csv` follow these naming patterns (set by
`pipeline._process_field` which prefixes with `{channel}_`):

- **Patch metrics**: `{channel}_patch_{MetricName}_{stat}` where stat is mean/min/max
  - e.g. `nuclei_patch_VarianceLaplacian_mean`, `nuclei_patch_VarianceLaplacian_min`
- **Global metrics**: `{channel}_{metric_name}`
  - e.g. `nuclei_power_log_log_slope`, `nuclei_global_variance_laplacian`

The exact columns depend on the user's `quality` config (which channels, which metrics).
Rather than hardcoding metric names, **auto-discover** focus columns from the CSV by
matching known prefixes/patterns.

#### Aggregation strategy (per well)

All focus metrics are quality scores (not counts), so **averaging across fields** is the
right aggregation. Use `aggregate_per_well(df, col, "mean")`.

- `_mean` columns в†’ mean of means (correct for well-level summary)
- `_min` columns в†’ mean of per-field minimums (shows typical worst patch per well)
- `_max` columns в†’ mean of per-field maximums
- Global metrics в†’ mean across fields

#### Implementation plan

1. **Add focus heatmap generation inside `generate_all_heatmaps()`** in `heatmaps.py`
   - Auto-detect focus columns: any column containing `_patch_` or matching known global
     metric names (`power_log_log_slope`, `global_variance_laplacian`, `high_freq_ratio`)
   - For each detected column, generate a heatmap with `agg_func="mean"` and save as
     `focus_{column_name}.html`
2. **No config changes needed** вЂ” the set of focus metrics is already defined by
   `quality.compute_on`, `quality.patch_metrics`, `quality.global_metrics`; the CSV
   columns reflect whatever was computed
3. **Tests**: synthetic CSV with focus columns в†’ verify heatmap files are generated

### 8. Bug: n_nuclei != n_cells in CSV вЂ” investigate and fix

**Symptom**: `n_nuclei` and `n_cells` don't match in `field_measurements.csv`. After
`_match_cells_to_nuclei` and `relabel_consecutive`, they should be identical because
every cell gets a nucleus ID and vice versa.

#### Root cause analysis

The bug is in `CellposeSegmenter._match_cells_to_nuclei` (cellpose.py:100-133).

When **two nuclei overlap with the same Cellpose cell**, the second nucleus overwrites
the first:

```python
for nuc_id in np.unique(nuclei_labels):    # sorted order
    ...
    best_cell = np.bincount(overlapping).argmax()
    output[cell_labels == best_cell] = nuc_id   # в†ђ overwrites if same cell
```

Example: nuclei 1 and 2 both overlap most with Cellpose cell X.
1. Iteration nuc_id=1: `output[cell==X] = 1`
2. Iteration nuc_id=2: `output[cell==X] = 2` в†’ **overwrites**, cell ID 1 disappears

After the loop, nucleus 1 exists in `nuclei_labels` but has no matching cell in
`cell_labels`. Then `count_labels()` reports n_nuclei > n_cells.

This happens when Cellpose under-segments (merges two nuclei into one cell body),
which is common for tightly packed cells.

#### Proposed fix

In `_match_cells_to_nuclei`, track which Cellpose cells have already been claimed.
When a cell is already taken by a previous nucleus, fall back to using the nucleus
mask as the cell (same as the "no overlap" branch). This ensures every nucleus gets
exactly one cell:

```python
claimed_cells = set()
for nuc_id in np.unique(nuclei_labels):
    if nuc_id == 0:
        continue
    nuc_mask = nuclei_labels == nuc_id
    overlapping = cell_labels[nuc_mask]
    overlapping = overlapping[overlapping > 0]

    if len(overlapping) > 0:
        best_cell = np.bincount(overlapping).argmax()
        if best_cell not in claimed_cells:
            output[cell_labels == best_cell] = nuc_id
            claimed_cells.add(best_cell)
        else:
            # Cell already claimed вЂ” use nucleus mask as cell
            output[nuc_mask] = nuc_id
    else:
        output[nuc_mask] = nuc_id
```

This guarantees n_cells == n_nuclei after matching.

#### Done (2026-03-04)
- [x] Rewrote `_match_cells_to_nuclei` with greedy matching by nucleus occupancy fraction
  (score = overlap_pixels / nucleus_area). All (cell, nucleus) pairs sorted by score
  descending; greedily assigned вЂ” each cell and nucleus matched at most once. Unmatched
  cells (no nucleus) and unmatched nuclei (no cell) dropped. Better than the proposed
  `claimed_cells` approach: principled metric, avoids first-come bias.
- [x] Unit tests added in `tests/unit/test_cell_segmentation.py`

### 9. Enforce 1:1 nucleusв†”cell correspondence вЂ” remove orphans

**Goal**: After all post-processing, guarantee that every nucleus ID has a matching cell
ID and vice versa. No orphan nuclei, no orphan cells.

#### Where orphans can arise

1. **`_match_cells_to_nuclei`** (Task 8 bug): Two nuclei claim the same Cellpose cell в†’
   one nucleus loses its cell. Task 8's `claimed_cells` fix addresses this, but as a
   safety net we should also clean up afterwards.

2. **`remove_border_objects`**: Removes cells touching the border and nuclei with the
   same ID. This is correct IF IDs match perfectly. But if any ID mismatch crept in
   earlier (e.g. a cell ID that exists without a nucleus), border removal won't catch it.

3. **Cellpose under/over-segmentation edge cases**: Cellpose could produce cells that
   don't overlap with any nucleus at all (cell without nucleus). The current matching
   code only iterates over nuclei, so any Cellpose cell that no nucleus claims simply
   vanishes from the output. This is actually correct вЂ” but worth verifying.

#### Proposed approach

Add a `remove_orphans(nuclei_labels, cell_labels)` function to `postprocessing.py`:

```python
def remove_orphans(nuclei_labels, cell_labels):
    """Zero out IDs that exist in one array but not the other."""
    nuc_ids = set(np.unique(nuclei_labels)) - {0}
    cell_ids = set(np.unique(cell_labels)) - {0}

    # Nuclei without cells
    for orphan in nuc_ids - cell_ids:
        nuclei_labels[nuclei_labels == orphan] = 0

    # Cells without nuclei
    for orphan in cell_ids - nuc_ids:
        cell_labels[cell_labels == orphan] = 0

    return nuclei_labels, cell_labels
```

Call it in `_process_field()` right after `remove_border_objects` and before
`relabel_consecutive`:

```
cell_labels, nuclei_labels = remove_border_objects(...)
nuclei_labels, cell_labels = remove_orphans(nuclei_labels, cell_labels)  # NEW
nuclei_labels, cell_labels = relabel_consecutive(...)
```

#### Done (2026-03-04)
- [x] Unmatched nuclei zeroed in-place inside `CellposeSegmenter.segment()` (no separate
  `remove_orphans` function needed вЂ” the matching itself drops orphans)
- [x] Removed `relabel_consecutive` call from `pipeline.py` (no longer needed)
- [x] Test: `test_segment_zeros_unmatched_nuclei_inplace` in `test_cell_segmentation.py`
